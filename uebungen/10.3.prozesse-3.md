## Übung 12: Prozesse gezielt finden und steuern

In dieser Aufgabe sollst du Prozesse **ohne Nutzung von `ps`** kontrollieren.

1. Starte mehrere Prozesse:
   ```bash
   sleep 300 &
   sleep 400 &
   sleep 500 &
   ```

2. Ermittle **alle laufenden `sleep`-Prozesse**, ohne `ps` zu verwenden.

3. Gib die gefundenen PIDs aus und prüfe, wie viele Prozesse laufen.

4. Halte **alle** diese Prozesse mit einem einzigen Kommando an.

5. Prüfe mit `jobs` und einem geeigneten Kommando, ob die Prozesse wirklich angehalten sind.

6. Beende anschließend **nur einen** der Prozesse sauber.

7. Beende danach die restlichen Prozesse auf einmal.

### Fragen:
- Warum ist die Steuerung über Prozessnamen in der Praxis hilfreich?
- Welche Risiken bestehen, wenn mehrere Prozesse denselben Namen haben?
- Wann ist ein gezieltes Beenden besser als ein pauschales?

## Übung 13: Prozessgruppen und Signalweitergabe

Ziel: Verstehen, dass Signale oft **an Prozessgruppen** gesendet werden.

1. Starte in einem Terminal folgendes Kommando:
   ```bash
   bash
   ```

2. Starte innerhalb dieser Shell:
   ```bash
   sleep 1000 | sleep 1000
   ```

3. Beende das Kommando mit:
   ```text
   STRG + C
   ```

4. Starte die Pipeline erneut:
   ```bash
   sleep 1000 | sleep 1000
   ```

5. Ermittle aus einem zweiten Terminal die PIDs der beiden `sleep`-Prozesse.

6. Beende **nur einen** der beiden Prozesse mit:
   ```bash
   kill -SIGTERM PID
   ```

7. Beobachte:
   - Läuft der andere Prozess weiter?
   - Wie verhält sich die Pipeline?

### Fragen:
- Warum beendet `STRG + C` beide Prozesse gleichzeitig?
- Warum beeinflusst `kill` nur einen Prozess?
- Was ist eine Prozessgruppe?
- Warum sind Prozessgruppen wichtig für die Terminalsteuerung?
